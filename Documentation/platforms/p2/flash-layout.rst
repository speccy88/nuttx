Flash layout
============

Status: STATICALLY-VERIFIED logical layout, HIL-REQUIRED enforcement, flash
boot, and filesystem.

This file is generated by ``tools/p2/generate-flash-layout.py`` from
``tools/p2/lib/flash_layout.py``.  The model reserves the 16 MiB W25-class
flash as follows:

* ``0x000000-0x07ffff``: logical boot reservation
  (512 KiB).  Runtime write protection is not implemented yet.
* ``0x080000-0xffffff``: logical filesystem
  reservation (15.5 MiB).  SmartFS binding is not
  implemented yet; media validation remains HIL-required.

The boundary follows the pinned ``loadp2 -SINGLE -FLASH`` implementation.
The ROM reads a 1 KiB boot window, but loadp2 places the first application byte
at ``0x000090`` inside that window and programs at least four
256-byte pages.  The P2 port permits at most ``0x7c000`` image bytes, so
programmed data ends by ``0x07c100``.  Because loadp2 uses
a 64 KiB erase whenever more than 64 program pages remain, the maximum erase
range is ``[0x000000, 0x080000)``.

``tools/p2/verify-flash-layout.py`` checks generated-file drift and prints
exact program/erase ranges for an input.  ``tools/p2/flash.sh`` requires an
explicit port, image, ``--execute``, ``P2_HIL=1``, and
``P2_ALLOW_FLASH_WRITE=1`` before invoking the pinned loader.  It also requires
``P2_ALLOW_SD_WRITE=1`` because the flash loader drives shared P60/P61 in a way
that can select and clock an installed microSD card.  Persistence, bootability,
JEDEC geometry, partition enforcement, and filesystem safety remain
HIL-required.
